---
title: Messages vs StateWeave
description: Compare transcript memory with immutable causal graph state.
---

Most agent runtimes store a transcript:

```txt
system → user → assistant → tool → assistant
```

Each call replays or summarizes that sequence.

StateWeave stores causal state:

```txt
system ─┐
        ├→ goal → tool_call → tool_result → resource
frontier┘                              └→ answer
```

## Stored truth

Traditional runtime:

```txt
messages[]
```

StateWeave:

```ts
type AgentState = {
  version: 1;
  nodes: CausalNode[];
  frontier: string[];
};
```

## Model input

StateWeave compiles a bounded working context from:

- the active frontier
- the current goal and system contract
- current resource heads
- recent authoritative evidence
- bounded operational ancestry
- relevant older nodes
- a deterministic whole-graph digest

The full source graph remains append-only. Projection can omit or supersede details without deleting them.

## Model output

The model returns one ordinary tool action or a final answer. StateWeave automatically records the action's exact compiled read set as causal parents.

This keeps graph structure deterministic and prevents graph-authoring syntax from becoming a second agent protocol.
