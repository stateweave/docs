---
title: Add tools
description: Define Zod-validated tools for the StateWeave Agent.
---

A tool has a name, description, schema, and executor.

```ts
import { z } from "zod";
import type { Tool } from "stateweave";

const lookup: Tool = {
  name: "lookup_order",
  description: "Look up one order by id.",
  schema: z.object({ orderId: z.string() }),
  async execute(args) {
    const { orderId } = z.object({ orderId: z.string() }).parse(args);
    return database.orders.get(orderId);
  }
};
```

Pass custom tools to `Agent`:

```ts
import { Agent } from "stateweave";

const agent = new Agent({ model, tools: [lookup] });
```

The model requests one action:

```txt
TOOL_CALL {"name":"lookup_order","args":{"orderId":"ord_123"}}
```

StateWeave validates the arguments, executes the tool, then appends:

```txt
tool_call → tool_result → resource/evidence
```

The next inference receives the relevant tool evidence through causal projection.

## Default workspace tools

If `tools` is omitted, `Agent` provides:

- `read_file`
- `write_file`
- `edit_file`
- read-only `bash_command`

Pass `tools` explicitly to replace the defaults.
