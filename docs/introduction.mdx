StateWeave replaces transcript-shaped agent memory with an immutable causal graph.

```txt
immutable causal graph → bounded working context → ordinary model action → causal graph
```

There is one public agent class: `Agent`.

The runtime automatically records goals, model inferences, tool calls, tool results, resource versions, protocol errors, and answers. Every model action points to the exact graph nodes compiled for that inference.

The model remains a normal transformer. It receives a temporary linear rendering because provider APIs require one, but StateWeave stores graph state—not `messages[]`.

## What this unlocks

- lossless append-only working state
- exact causal lineage for model actions
- bounded model context without deleting graph history
- current resource heads and superseded versions
- inspectable prompts, actions, tools, retries, and token metrics
- dependency-free graph visualization

## Runtime ownership

The model uses ordinary actions:

```txt
TOOL_CALL {"name":"read_file","args":{"file_path":"src/app.ts"}}
```

or:

```txt
FINAL: The parser is fixed and the focused check passes.
```

The runtime—not the model—creates graph nodes and edges.
